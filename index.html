Whats the difference? npm v. Bower

Types of Use

  • npm is for managing Node Modules but can be used for Front End "Modules"
  • npm Utilizes nested dependency tree for ease of inclusion - A.K.A. "need-to-know"
  • Bower is mainly for front-end tools and works across many worlds like Grails, Ruby, Node, PHP, Etc.
  • Bower requires a flat dependency tree for ease of selection

Dev Workflow Tools

Grunt

  • Configuration centric
  • Read Input -> process to output file
  • Read output -> process next task ->
  • Deposit final output for use
  • Non threaded

Gulp

  • Output centric - Piped i/o streams
  • Read input ->
  • pipe stream through connected tasks->
  • Deposit output
  • Piped Streams (threaded)

npm Install some things


  npm install -g eslint gulp gulp-cli grunt grunt-cli mocha
  • Grunt, Gulp, Mocha and eslint are tools which offer command line interface.
  • Installing globally adds them to the shell path so they can be called directly.
  • npm 1.0: Global vs Local installation

For instance, eslint can be installed globally for Atom to use.

Gulp tasks can be called directly once setup; i.e. 'gulp build', to manually run and verify a gulp build task, or 'mocha test' to run a test suite directly.

General, the rule of thumb:

  • If you’re installing something that you want to use in your program, using require('whatever'), then install it locally, at the root of your project.
  • If you’re installing something that you want to use in your shell, on the command line or something, install it globally, so that its binaries end up in your PATH environment variable. 

  npm install express gulp gulp-cli gun --save

'--save' tools that you expect to need for final run-time. 


  npm install gulp-nodemon gulp-eslint gulp-jshint gulp-sass --save-dev
  npm install browser-sync jshint jshint-stylish --save-dev
  npm install mocha chai --save-dev

--save-dev tools which are mainly for development use only like live reload and pre-compilers and sourcemaps. Also include tools like gulp or eslint again in case the end user doesn't have them installed. 

npm install gulp-nodemon gulp-eslint gulp-jshint gulp-sass browser-sync jshint jshint-stylish mocha chai --save-dev

 {

 }]
}
},
...
grunt.loadNpmTasks( 'grunt-contrib-copy' );

Edit Lines in gruntfile.js

Add 'copy' to the Default and Serve Task dependency array as the first dependency 

// Default task
grunt.registerTask( 'default', [ 'copy', 'css', 'js' ] );
// Serve presentation locally
grunt.registerTask( 'serve', [ 'copy', 'connect', 'watch' ] );
